### Page-scoped JS bulk delete

`JScript/GUI/*.js` — every file here matches a `GUI/*.aspx` page. After ASPX delete, the JS becomes orphaned. Bulk delete:

```bash
# In legacy repo, AFTER ASPX delete:
find JScript/GUI -name "*.js" | while read f; do
  page=$(basename "$f" .js)
  if [ ! -f "GUI/${page}.aspx" ] && [ ! -f "GUI/${page}V2.aspx" ]; then
    echo "Safe to delete: $f"
  fi
done
```

## Rollback safety

Each step (M-090 → M-091 → M-092) is reversible via git. **Recommended sequence**:

1. Branch from master: `cleanup/p7-decommission`
2. Run M-089 audit script → commit results as `docs/DECOMMISSION_AUDIT.md`
3. Run M-090 deletes → 1 commit per page group (User Mgmt, Agency Mgmt, etc) for easy revert
4. Run M-091 lib deletes → 1 commit per lib (bootstrap-3, bootstrap-4, font-awesome, animate)
5. Run M-092 scss cleanup → 1 commit
6. Open PR → squash-merge after backend team confirms reverse proxy stability for full sprint
7. Tag legacy repo `v-final-aspx-archive` before PR squash for permanent history pointer
